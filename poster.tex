\begin{block}{Motivation}
\begin{itemize}
\item DeepSeek-V3 MoE: 256 routed experts (32 local/device), top-8 routing, hidden dim 7168, intermediate dim 2048
\item Two FP8 block-scaled GEMMs per forward pass on NVIDIA B200: ${\sim}2{,}500$ TFLOPS FP8, 96\,MB L2, 8\,TB/s HBM3e
\item Na\"ive per-expert loop wastes hardware: kernel launch overhead dominates small batches; atomic scatter hurts large batches
\item \textbf{Goal:} A single fused Triton kernel covering $T{=}1$ to $T{=}14{,}107$ tokens without framework-level dispatch overhead
\item Prior work (MegaBlocks, vLLM) relies on grouped GEMM or per-expert loops; we fuse all stages into a monolithic pipeline with workload-adaptive tiling
\end{itemize}
\end{block}

\begin{block}{Six-Stage Pipeline}
\begin{center}
\includegraphics[width=0.95\linewidth]{figures/pipeline.pdf}
\end{center}
\begin{enumerate}
\item \textbf{Routing:} Triton DeepSeek-V3 no-aux routing (sigmoid, group top-4, global top-8, normalization)
\item \textbf{Sorting:} Tile-parallel histogram with atomic offsets; hybrid CPU path for $T{<}1024$
\item \textbf{GEMM1+SwiGLU:} Fused $W_1$/$W_3$ projections in shared K-loop with \texttt{tl.dot(fp8,fp8)}
\item \textbf{GEMM2:} Non-atomic down-projection $\to$ \texttt{expert\_out} (bf16)
\item \textbf{Token Reduce:} 2D tiled accumulation of $K{=}8$ expert contributions per output token
\item \textbf{$T{=}1$ path:} Fully fused routing+sort+GEMM for single-token decode
\end{enumerate}
\end{block}

\begin{block}{Experimental Setup}
\begin{itemize}
\item \textbf{Hardware:} NVIDIA B200 (Blackwell, sm\_100a)
\item \textbf{Software:} PyTorch 2.12.0+cu132, Triton 3.7.0
\item \textbf{Scoring:} CUPTI GPU kernel time only --- CPU overhead (${\sim}600\,\mu$s, 56--73\% of wall time) excluded
\item \textbf{Tolerance:} $\text{atol}{=}1.0$, $\text{rtol}{=}0.3$, 90\% ratio (${\sim}100\times$ wider than our $\text{atol}{=}0.01$)
\item 19 real-trace workloads, 15 warmup + 80 iters
\item All claims: same-session A/B (${\pm}2\%$ mean floor)
\item Baseline: FlashInfer reference kernel (per-expert GEMM loop)
\end{itemize}
\end{block}

\begin{block}{Key Optimizations}

{\large\bfseries 1. FP8 Native Tensor Core Dot \hfill $\boldsymbol{2\times}$ throughput}
\vspace{2pt}

Blackwell \texttt{tcgen05.mma} supports native FP8$\times$FP8 dot with FP32 accumulation.
Block-scale dequantization \emph{after} the dot:

\vspace{2pt}
\texttt{~~partial = tl.dot(a, b) ~~\# fp8 $\times$ fp8}\\
\texttt{~~acc += partial * a\_scale * b\_scale}
\vspace{4pt}

Both $W_1$/$W_3$ projections fused in shared K-loop, amortizing A-side loads, followed by inline SwiGLU.

\vspace{4pt}\rule{\linewidth}{0.3pt}\vspace{4pt}

{\large\bfseries 2. Non-Atomic GEMM2 + Token Reduce \hfill $\boldsymbol{+46\%}$}
\vspace{2pt}

Na\"ive \texttt{tl.atomic\_add} scatter = $6\times$ BW amplification (12\,B FP32 RMW vs.\ 2\,B bf16 store). Two-pass:
\begin{itemize}
\item GEMM2 writes bf16 \texttt{expert\_out} (non-atomic, zero-free)
\item Token reduce reads exactly 8 contributions, accumulates in FP32
\end{itemize}

\vspace{4pt}\rule{\linewidth}{0.3pt}\vspace{4pt}

{\large\bfseries 3. FP16 Intermediate ($\times$0.125/$\times$8.0) \hfill $\boldsymbol{+4.5\%}$}
\vspace{2pt}

Scale SwiGLU by $\times$0.125 in GEMM1 epilogue (range $\to {\pm}5{,}000$, within FP16 max 65{,}504) $\to$ FP16 store $\to$ $\times$8.0 in GEMM2. Halves HBM traffic. FP32 fallback for $T{<}32$.

\vspace{4pt}\rule{\linewidth}{0.3pt}\vspace{4pt}

{\large\bfseries 4. Bucket Specialization (4 tiers) \hfill $\boldsymbol{88\times \to 107\times}$}
\vspace{2pt}

BLOCK\_M = 16/32/64/128. Column-major dispatch (GROUP\_M=32/64): consecutive blocks process different tokens of the \emph{same} expert, maximizing L2 weight reuse.

\vspace{4pt}\rule{\linewidth}{0.3pt}\vspace{4pt}

{\large\bfseries 5. Compiler Hints \& Cache Policy \hfill $\boldsymbol{+4.9\%}$}
\vspace{2pt}

Disable LLVM LSR (avoids register spills in SwiGLU addressing). Asymmetric L2 eviction: weights \texttt{evict\_last} (high reuse), tokens \texttt{evict\_first} (streaming).

\vspace{4pt}\rule{\linewidth}{0.3pt}\vspace{4pt}

{\large\bfseries 6. Deep Pipeline + 2D Reduce \hfill $\boldsymbol{+5\%}$}
\vspace{2pt}

GEMM2 \texttt{num\_stages=6--8} hides HBM latency (+2.1\% mean). 2D tiled token reduce ($[\text{BLOCK\_T}, \text{BLOCK\_N}]$) cuts grid $>$10$\times$, enables ILP (+2.9\% large-$T$).

\end{block}

\begin{block}{Precision Hierarchy}
\begin{center}
\includegraphics[width=0.85\linewidth]{figures/precision_matrix.pdf}
\end{center}
\textbf{Intermediate} (GEMM1$\to$GEMM2): requires $\ge$FP16. SwiGLU spans ${\pm}40$K, exceeding fp8 max (448); bf16 passes 8/19 only. Scaling $\times$0.125/$\times$8.0 clamps to ${\pm}5$K (fp16-safe), halving HBM traffic.

\textbf{Expert output} (GEMM2$\to$reduce): bf16 OK (range $3.4{\times}10^{38}$, no overflow); fp16 overflows 3/19. bf16 saves 50\% write BW on $[\text{MAX\_PADDED}, 7168]$ (+0.4\%). Contest $\text{atol}{=}1.0$ accommodates bf16 precision loss.

\textbf{MXFP8} (\texttt{tl.dot\_scaled}, UE8M0): matched ratio 0.17--0.32. Shared exponents across 128-element blocks too coarse for the wide, heterogeneous value distribution of SwiGLU outputs.
\end{block}

\begin{block}{Key Takeaways}
\begin{enumerate}
\item \textbf{Understand scoring:} CUPTI GPU-only timing hides ${\sim}600\,\mu$s CPU overhead---don't optimize what isn't measured
\item \textbf{Precision $\ne$ spectrum:} FP16 passes 19/19 (+4.5\%); bf16 fails 11/19; fp8 fails all---mantissa bits matter more than range
\item \textbf{A/B test everything:} $\pm$15\% per-workload noise demands same-session comparison ($>$2\% threshold)
\item \textbf{Profile first:} All 8 successful techniques identified via NCU profiling, not speculation
\item \textbf{Tolerance matters:} Contest $\text{atol}{=}1.0$ unlocks bf16 expert output ($+0.4\%$ mean) that fails strict $\text{atol}{=}0.01$
\end{enumerate}
\end{block}

\begin{block}{Future Work}
\begin{itemize}
\item Small-$T$: 5.6\% of FP8 peak at $T{=}7$ (expert weight cold misses dominate)
\item L2 persistent cache (80\,MB GEMM2 weights) and epsilon expert skipping: prototyped but disabled
\item MXFP8 \texttt{tl.dot\_scaled}: matched\_ratio = 0.17--0.32 (UE8M0 shared exponents too coarse)
\item Workload-adaptive kernel switching for production MoE serving
\item The GEMM1$\to$GEMM2 bottleneck crossover suggests dynamic per-workload selection is essential
\item Extend to multi-GPU expert parallelism with all-to-all communication overlap
\end{itemize}
\end{block}

\begin{block}{Measurement Noise}
Modal B200 shared instances: ${\pm}2\%$ mean noise, ${\pm}15\%$ per-workload for small $T$ ($<$0.2\,ms kernels), $<$1\% for large $T$. Cross-session drift ${\pm}20{-}30\%$ from co-tenant interference. We adopt a 2\% mean threshold: only exceeding optimizations retained. Contest tolerance ($\text{atol}{=}1.0$) enables bf16 expert output (abs errors ${\sim}600$K) that fails strict validation.
\end{block}
